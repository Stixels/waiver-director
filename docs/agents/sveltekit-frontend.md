# Svelte And Route Boundaries

- Follow the existing Svelte 5 runes-mode patterns already used in this repository.
- Keep route boundaries clear: public marketing and auth flows, protected workspace app flows, and public waiver-link flows.
- Keep sensitive business logic and authorization in trusted server or backend layers.
- Favor small, composable components and scoped styling over one-off rewrites.
- Preserve accessibility semantics for forms, interactive controls, and keyboard navigation.
